# Session Viewer

这是一个用于将 Codex 本地 Session 存储的调用与对话记录转换为多种文本格式的 Node.js 工具。它包含了命令行批量转换工具，以及一个基于 Web 的前端可视化分析面板，用于深度分析、浏览和管理 Codex 的会话日志。

---

## 前端 Web 分析面板功能

### 1. 左右分栏气泡对话流
- 用户提问：自动分析提取用户提问时的 IDE 上下文，包括活动文件与已开标签页，并以代码胶囊徽章呈现。
- Codex 回复：将 AI 的思考过程、工具执行链路与最终回答进行分离并靠左对齐，形成直观的对比。

### 2. 双重浏览视图
- 聚焦单轮：屏幕中仅展示当前选中的这一轮对话，包括用户提问、思考、工具调用与最终回答。支持通过键盘左方向键与右方向键切换。
- 时间线滚动：切换回纵向连续滚动的完整对话流。

### 3. 纯净对话流
- 开启后在时间轴上屏蔽思考过程和工具执行日志，仅留下人机对话。
- 提供显示 IDE 上下文复选开关，支持实时显隐用户卡片中的活动文件和标签页。
- 提供一键复制纯净对话文本按钮。

### 4. 纸带式终端
- 将命令行执行参数与终端输出整合入可折叠的控制台面板中。
- 在浅色模式下呈现暖纸灰色背景与暗色字迹，在深色模式下适配为暗色背景。

### 5. 联动彻底删除本地原始日志
- 侧边栏包含垃圾桶按钮，点击会唤起确认弹窗，并提供同时一键彻底删除本地原始 .jsonl 日志文件选项。
- 勾选后，系统除了清空隔离的 outputs 文件夹，还会自动在本地 sessions 工作区目录内进行深度多层递归查找，安全擦除对应的原始日志文件。

### 6. 一键拖拽即时解析与下载
- 浏览器中直接拖入全新的 .jsonl 文件进行即时解析并刷新侧边栏。
- 顶部元数据区支持直接下载干净版与极简版纯文本对话文档。

## 目录结构说明

```
Session-Viewer/
├── package.json         # 项目配置文件
├── README.md            # 项目使用与架构说明文档
├── .gitignore           # 忽略大体积原始日志与生成的文本文档
└── src/
    ├── index.js         # 核心协调器，兼容 CLI 与 Server 运行
    ├── config.js        # 配置解析、路径计算与文件系统验证
    ├── server.js        # RESTful API 服务，实现路径自动解码与安全删除
    ├── utils/
    │   ├── formatters.js    # 文本截断与时间戳格式化工具函数
    │   └── messageParser.js # 提取 IDE 上下文及清洗用户提问正文
    ├── parser/
    │   └── logParser.js     # 流式读取日志，提取时间线事件并聚合 Turn
    ├── public/          # 前端静态资源托管目录
    │   ├── index.html   # 分析面板主界面
    │   ├── style.css    # CSS 样式表
    │   └── app.js       # 前端控制逻辑
    └── writers/
        ├── dialogueWriter.js  # 写入 Clean 和 Super Clean 对话日志
        ├── historyWriter.js   # 写入包含工具调用的 verbose 轨迹文件
        └── splitTurnsWriter.js# 写入分轮次文件的专属写入器
```

## 使用指南

### 1. 运行 Web 服务器

启动后端 Web 服务：

```bash
npm run server
```

或者使用 node 启动：

```bash
node src/index.js --server
```

启动后在浏览器中打开 http://localhost:3000/ 即可访问可视化面板。

### 2. 命令行批量转换

如果您仅需要将特定 .jsonl 快速转换为纯文本，运行转换脚本并传入目标文件路径：

```bash
npm run convert -- <path_to_jsonl_file>
```

或者直接 node 执行：

```bash
node src/index.js <path_to_jsonl_file>
```

#### CLI 自动探测机制
如果您没有指定任何路径，命令行工具会执行以下策略：
1. 首先在当前目录下寻找默认预设的 rollout 文件。
2. 若不存在，将自动扫描当前工作目录，默认挑选体积最大的 .jsonl 文件进行即时转换。

## 二次开发与扩展

项目代码高度解耦，您可以随时在以下位置横向扩展功能：
- 引入新事件：如要解析新的日志事件，调整 src/parser/logParser.js。
- 自定义导出样式：如需新增 HTML、Markdown 或 PDF 格式文件，在 src/writers/ 下建立新的写入器，并在 src/index.js 中注册调用。
- 扩展 IDE 上下文变量：如需提取额外的编辑器状态参数，修改 src/utils/messageParser.js。
